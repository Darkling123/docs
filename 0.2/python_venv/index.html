      print_page_title: 'Print Site'

MKDOCS - versioning (mike)

? pip install mike
Edit mkdocs.yml and add the following 

plugins:
  - mike:
      # These fields are all optional; the defaults are as below...
      alias_type: symlink
      redirect_template: null
      deploy_prefix: ''
      canonical_version: null
      version_selector: true
      css_dir: css
      javascript_dir: js

Now use the following

mkdocs gh-deploy

? mike delete --all
? mike deploy 0.1
? mike set-default 0.1 ? mike serve --dev-addr 0.0.0.0:8000 &

MKDOCS - nginx (under python venv : myenv)

  1. Install Nginx (if not already installed):
    ? sudo apt update
    ? sudo apt install nginx

  2. Configure Nginx to Serve the MkDocs Site:
    Create a new configuration file for your site, e.g., /etc/nginx/sites-available/my_project
    ? nano default
    Modify to the following 

server {
  listen 80 default_server;
  listen [::]:80 default_server;

  server_name articles123.kozow.com;

  root /var/www/html;

  index index.html index.htm index.nginx-debian.html;

    location / {
      try_files $uri $uri/ =404;
    }
}

  1. enable the site by creating a symbolic link to sites-enabled:
    ? sudo ln -s /etc/nginx/sites-available/my_project /etc/nginx/sites-enabled/

  2. Restart Nginx to apply the changes:
    ? sudo systemctl restart nginx

  3. Enable HTTPS with a TLS/SSL Certificate
    Install Certbot: Certbot is a tool to automate the installation of Let's Encrypt certificates.
    ? sudo apt install certbot python3-certbot-nginx

  4. Using dynu.com for dynamic DNS
    Obtain an SSL Certificate for articles123.kozow.com: Use Certbot to obtain an SSL certificate for your Dynu Dynamic DNS hostname:
    ? sudo certbot --nginx -d articles123.kozow.com
    email address : darkling123@hotmail.com

OPTION : Could install Firewall; However will use the router to do this.
