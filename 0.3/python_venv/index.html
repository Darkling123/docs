2.4 MKDOCS - versioning (mike)

Inside the following location /NETWORKDRIVE/James/04_SYSTEMS/debian/
? pip install mike
Edit mkdocs.yml and add the following 

plugins:
  - mike:
      # These fields are all optional; the defaults are as below...
      alias_type: symlink
      redirect_template: null
      deploy_prefix: ''
      canonical_version: null
      version_selector: true
      css_dir: css
      javascript_dir: js

Now use the following

? mkdocs gh-deploy ? mike delete --all

The following ? mike deploy 0.1
? mike set-default 0.1 ? mike serve --dev-addr 0.0.0.0:8000 &

Noticed that any immediate changes don't show up automatically anymore on the served web page. Therefore can use the following command to update the immediate version However, noting this will likely impact the cucrrent version, look to update on a new version. mike deploy --update 0.1

2.8 Make a script to run the mike deploy and also backup the yml and markdown files on the network drive

Location is ~/NETWORKDRIVE\James\04_SYSTEMS\debian\mkdocs Script is 

#!/bin/bash

pkill mkdocs
pkill mike

# Get the version as the first argument
VERSION=$1

# Define the backup directory
BACKUP_DIR="backups/$VERSION"

# Create the backup directory for the current version (if it doesn't exist)
mkdir -p $BACKUP_DIR

# Copy the mkdocs.yml file to the backup directory, forcing overwrite
cp -f mkdocs.yml $BACKUP_DIR/

# Copy all Markdown files (assu ming they are in the docs/ directory) to the backup directory, forcing overwrite
cp -rf docs/ $BACKUP_DIR/

# Deploy the new version with mike
mike deploy $VERSION

echo "Backup and deployment of version $VERSION completed."

? ./backup_and_deploy.sh v0.2

2.9 MKDOCS - nginx (under python venv : myenv)

  1. Install Nginx (if not already installed):
    ? sudo apt update
    ? sudo apt install nginx

  2. Configure Nginx to Serve the MkDocs Site:
    Create a new configuration file for your site, e.g., /etc/nginx/sites-available/my_project
    ? nano default
    Modify to the following 

server {
  listen 80 default_server;
  listen [::]:80 default_server;

  server_name articles123.kozow.com;

  root /var/www/html;

  index index.html index.htm index.nginx-debian.html;

    location / {
      try_files $uri $uri/ =404;
    }
}

  1. enable the site by creating a symbolic link to sites-enabled:
    ? sudo ln -s /etc/nginx/sites-available/my_project /etc/nginx/sites-enabled/

  2. Restart Nginx to apply the changes:
    ? sudo systemctl restart nginx

  3. Enable HTTPS with a TLS/SSL Certificate
    Install Certbot: Certbot is a tool to automate the installation of Let's Encrypt certificates.
    ? sudo apt install certbot python3-certbot-nginx

  4. Using dynu.com for dynamic DNS
    Obtain an SSL Certificate for articles123.kozow.com: Use Certbot to obtain an SSL certificate for your Dynu Dynamic DNS hostname:
    ? sudo certbot --nginx -d articles123.kozow.com
    email address : darkling123@hotmail.com

OPTION : Could install Firewall; However will use the router to do this.
